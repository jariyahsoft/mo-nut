## Instructions

1. Confirm provisional brand tokens and implement typography, color, spacing, shape, icon, form, card, feedback, and status primitives.
2. Build mobile bottom navigation, safe-area/standalone/browser layouts, tablet master-detail, and desktop side navigation without duplicating feature logic.
3. Meet WCAG 2.2 AA, 44×44 CSS px targets, keyboard/focus/screen-reader, 200% zoom, reduced motion, Thai localization, and common data states.

## Verify
- Component tests and accessibility scan pass
- Keyboard reaches every shell action
- Mobile/tablet/desktop layouts have no clipped critical content
- Status is never color-only

## Definition of Done
- [ ] Design tokens/components are reusable
- [ ] PWA shell is responsive
- [ ] Accessibility baseline passes
- [ ] Story/demo documentation exists
